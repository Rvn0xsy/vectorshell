# AGENT.md

## Project

VectorShell

VectorShell is an AI-driven remote command execution platform written in Rust.

The system consists of:

* VectorShell Server
* VectorShell Client

Clients connect to the server using **reverse WebSocket connections**.

The **server hosts the AI agent** which generates commands and orchestrates remote execution.

Clients **only execute commands and return results**.

---

# Core Principles

AI agents implementing this repository must follow these rules:

1. **The server controls all execution**
2. **Clients do not decide commands**
3. **Clients contain no AI logic**
4. **All communication uses structured JSON**
5. **Clients are generated by the server**
6. **Clients contain embedded configuration**

Clients must not require CLI arguments.

# Server Configuration

The server must load configuration from:

```
config/config.toml
```

Example:

```
[server]
listen = "0.0.0.0:8080"
ws_path = "/ws"

[agent]
model = "gpt-4o"

[client]
default_server = "wss://127.0.0.1:8080/ws"
reconnect_interval = 5
insecure_tls = false

[auth]
token = "vectorshell-secret"
```

Server startup:

```
vectorshell-server --config config/config.toml
```

---

# Client Generation

Clients must be generated by the server.

Command:

```
vectorshell-server generate-client
```

This command builds a client binary with embedded configuration.

The client must contain:

* server websocket address
* authentication token
* reconnect interval
* insecure TLS flag

These values are embedded at **compile time**.

Current generated envs include:

* `VECTOR_SERVER_URL`
* `VECTOR_AUTH_TOKEN`
* `VECTOR_RECONNECT_INTERVAL`
* `VECTOR_INSECURE_TLS`

Changing config requires regenerating client binaries.

---

# TLS and Proxy Notes

Server supports both `ws://` and `wss://`.

For `wss://`, configure `[tls]` in `config/config.toml`.

Windows generated clients support system proxy discovery:

1. WinHTTP auto proxy (PAC/WPAD)
2. Manual proxy from Internet Settings

If a proxy is found, client uses HTTP CONNECT tunnel before WebSocket handshake.

---

# Embedded Client Configuration

Client configuration must be embedded using compile-time constants.

Example:

```
pub const SERVER_URL: &str = env!("VECTOR_SERVER_URL");
pub const AUTH_TOKEN: &str = env!("VECTOR_AUTH_TOKEN");
```

These values are injected by the server when compiling the client.
